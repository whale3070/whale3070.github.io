Polkadot-黑客松-demoday分析

1. 送你一朵小红花-慈善项目

这份录音转写记录了 2026年1月11日 举办的一场 Demo Day 活动。首支展示的团队名为 “送你一朵小红花”，由队长苏（Su）带领，参与的是 XCM（波卡跨链） 赛道。

以下是对该项目及现场表现的深度分析：

1. 项目核心：去中心化慈善平台

该项目旨在利用波卡（Polkadot）生态构建一个去中心化、透明且具有隐私保护的慈善捐助平台 。

*
技术定位：不直接做公益项目，而是定位为技术基础设施，为机构提供智能合约和跨链能力 。

*
核心痛点：解决 Web2 慈善资金流向不透明、跨境捐赠困难、中介抽成及隐私泄露等问题 ；同时解决 Web3 现有产品跨链操作复杂、Gas 费高及可持续性差的现状 。

  • 技术亮点：

跨链能力：基于波卡 SDK 和 ISMP 协议，支持以太坊、BSC 等 EVM 系链的随手捐赠 。

*
隐私保护：接入 ZK（零知识证明） 技术保护捐赠者和受助者的身份隐私 。

  • 激励机制：引入“小红花”概念，既是链上的荣誉勋章（SBT/凭证），也是实际的经济奖励 。

2. 商业模式与可持续性

该团队在预判行业生存问题上表现得非常务实，这与你提到的“排除错误选项”逻辑高度契合 。

*
盈利模式（To B）：明确拒绝从捐款中抽成（确保善款近乎 100% 到达受助者），转而向公益机构收取订阅费、协议费、跨链费或年费 。

  • 市场对标：参考了波卡已有的慈善捐赠数据（累计 3 亿+ 捐赠量），认为提供 B2B 服务模型是可行性极高且能存活的路 。

3. 现场 Demo 与技术反馈

*
开发进度：已完成平台网站、捐赠中心、公示墙及合约上链。目前已接入 ISMP 协议，但 XCM 跨链因技术影响尚未完全实现 。

*
评委建议：评委指出，由于项目未来可以直达 Parachain（平行链），建议可能没必要非得强求 XCM，给出了更直接的技术路径优化方案 。

  • AI 工具赋能：队长提到在开发过程中使用了 AI 编程工具，直观感受开发效率提升了 60% 以上，尤其是在学习新技术和辨识代码正确性方面 。

4. 关键洞察与预判（结合你的背景）

*
破除权威与独立思考：你在小学时就通过纠正老师逻辑破除了对权威的迷信。在此项目中，该团队也表现出不迷信“理想主义（用爱发电）”，而是清醒地寻找可持续的商业路径 。

*
全球信息获取权：该团队强调利用 AI 跨越搜索瓶颈，这正是你预判中提到的“获取全球信息权”的实战体现 。

  • 关于未来扩展：在回答评委关于“如何与阿里等 Web2 巨头竞争/合作”时，队长提到利用 GEO（Generative Engine Optimization） 等 AI 相关技术手段来提升品牌曝光，而非传统的搜索引擎优化 。

2. ML trader

以下是针对该项目的核心逻辑、技术实现及行业洞察的分析：

1. 项目核心：LLM 驱动的“盘感”交易

该团队认为传统的量化交易过度依赖“特征工程”和“线性回测”，虽然逻辑清晰，但在处理市场波动中的“非理性语义”时存在局限 。

  • 核心逻辑：利用大语言模型（LLM）处理 图像 Token（K线图、盘面走势图） 和文本信息。他们认为图像蕴含的语义容量远大于文本，AI 能捕捉到传统量化模型无法感知到的“盘感” 。

  • 产品形态：一个集成 AI 预测分析、策略设置（止损、盈亏点）和链上自动执行的交易平台 。

2. 技术实现与架构

*
后端语言：选用 Rust。理由是其具备健全的数据处理能力、高效的数据集读取库（如 Polars）以及强大的异步处理框架（Tokio） 。

*
模型接入：目前接入了 DeepSeek（D-Seek）等模型，输出标准的 JSON 格式 指令，用于触发链上交易 。

*
链上交互：在波卡（Polkadot）测试网上进行，支持通过 MetaMask（小狐狸钱包）进行代币转账和交易模拟 。

  • 数据回测：目前已实现历史数据回测展示，未来计划接入交易所（如币安、欧易）的实时粒子流数据 。

3. 商业模式与进化路径

  • 盈利预想：

订阅制：针对高级模型或特定 Agent 组合收取订阅费 。

  • 跟单系统：用户可以跟随表现优异的 AI Agent 进行实盘交易 。

  • AI 进化机制：

对抗学习：让不同个性的模型同台竞技，在对抗中通过经验学习 。

  • 人工反馈：参考“克隆文档”模式，当 AI 做出错误判断时，人工介入告诉 AI 为什么错，从而实现 Agent 的进化 。

4. 评委互动与关键挑战

*
稳定性挑战：评委指出仅靠 Prompt（提示词）很难保证交易系统的稳定性 。团队回应称未来会将“人类的特征工程”与“AI 提示词”打包成内置工具，降低用户的操作门槛 。

  • 波卡生态优势：团队看好波卡的安全性架构及“训练场”机制，认为这能帮助项目从测试网平滑过渡到主网 。

💡 结合你个人背景的深度观察

这份材料中有些点非常契合你之前提到的“预判”逻辑：

  1. 关于 AI 效率与生存力：虽然这是第二组，但他们同样强调了 AI 对新技术学习和开发的加速作用。这验证了你对**“全球信息获取权”和“黑客松生存力”**的预判。
  2. 关于权威与工具：该项目试图用 AI 的“语义理解”去挑战传统金融量化的“特征工程”权威。正如你小学时挑战数学老师的逻辑，这个团队也在尝试用更底层的逻辑（大模型语义）去重构交易这个细分领域。
  3. 关于“错误选项”：队长提到“传统交易金融的心跳比低下，导致必须做大量琐碎的清洗工作” 。这其实也是一种“排除错误路径”的过程，直接寻找更高效的 LLM 路径。

3. 村口协议

这份录音转写记录的是 2026年1月11日 Demo Day 的第三个展示项目 —— “村口协议 (Cunkou Protocol)”。

该项目通过一个非常接地气的比喻——“村口大树下的议事”，试图解决波卡（Polkadot）治理体系中长期存在的效率与信息不对称问题。

1. 核心痛点：波卡 OpenGov 的“治理冷淡”

项目方指出，目前的波卡治理模型（OpenGov）存在以下几个核心问题：

  • 门槛过高：参与者难以理解复杂的专业图表和技术术语。
  • 沟通碎片化：信息散落在各种“小道消息”中，缺乏实时、直面提案人的交流渠道。
  • 投票偏差：由于缺乏充分的信息沉淀和实时跟踪，导致投票行为往往是盲目的。
  • 结论：OpenGov 目前“缺一个下议院”，即一个前置的舆论场和讨论区。

2. 产品解决方案：村口协议

项目提出了一个名为“村口协议”的中间层，定位为 OpenGov 的前置舆情讨论站。

  • 经济模型 (V-DAO)：用户通过抵押代币（如 Best 或 DOT）无感铸造“V-DAO”，从而获得 VP（发言点数）。

  • 防止权力垄断：为了防止大户（鲸鱼）通过资金优势刷屏，VP 的生成遵循边际递减原则，确保不是有钱就能无限发言，维持表达的公平性。

  • 功能分区：

  • 滚动区：类似抖音直播间，24 小时实时讨论。

  • 精选区：根据点赞和消耗的 VP 点数，将高质量内容推上热门。

  • 闭环激励：议题结束后，参与者的“良言警句”可以被铸造成 NFT (MT)，标记在账户上作为荣誉凭证。

3. 项目价值预判

  • 对投资者：降低了信息获取成本，将复杂的治理讨论转变为低门槛的社交体验。
  • 对项目方：提供了一个“压力测试场”，在正式提交 OpenGov 提案前，可以在这里统一进行叙事、答疑和社区引流。
  • 最终目标：让国库（Treasury）资金的每一分流向都“有迹可循”，通过前置讨论确保决策的公正性。

💡 结合你个人背景的深度观察

这个项目与你的思维模型有几个非常有趣的交叉点：

  1. “村口”与“褶式 (The Fold / micro)”：你提到的“褶式”即 micro。这个项目正是将宏大的、复杂的波卡治理“折叠”到了微观的、具体的人人可参与的“村口”场景中，这是一种典型的微观叙事重构。
  2. 排除错误选项：你在书里想用那句话：“当我排除了所有错误选项，剩下的就是正确选项”。这个团队其实是在尝试排除“盲目投票”和“信息茧房”这两个错误选项，试图寻找那个通往“理性决策”的正确路径。
  3. 对权威的祛魅：波卡 OpenGov 本身是一个极高权威、高技术门槛的系统。这个项目试图通过“下议院”的模式，让普通用户也能通过低成本的方式参与博弈，这本质上也是一种对“精英治理”权威的挑战。

4. πX (派克斯)

这份录音转写（01月11日_5_智能规整）记录的是 Demo Day 的第四个项目 —— “πX (派克斯)”。该项目同样定位在 AI Agent 驱动的去中心化交易 (DEX) 赛道，但其侧重点在于**“极致的用户体验”和“多智能体（Multi-Agent）协作”**。

以下是对该项目的深度分析：

1. 核心理念：抹平 Web3 的门槛

队长在开场时通过三个问题（追求过程还是结果？相信 AI 还是自己？）明确了受众：那些相信 AI、追求赚钱结果、但觉得现有交易界面太复杂的普通用户 。

*
AI First (Natural Language)：用户无需填写复杂的表单或理解技术指标，只需通过自然语言（语音或文字）表达意图，由 AI 处理底层逻辑 。

*
Mobile First (移动优先)：针对目前 Web3 应用在手机端体验极差（需要频繁切换钱包确认）的痛点，πX 采用了自托管内置钱包，让操作像 Web2 应用一样丝滑 。

  • 面向结果：直接帮用户执行到最后一步（拿到钱），而不是让用户在过程中纠结 。

2. 技术架构：多智能体协作 (Multi-Agent)

该项目基于蚂蚁开源的 AntChain (或类似框架)，实现了三个核心 AI 角色协作 ：

1.
用户代理 (User Agent)：负责理解意图、维护对话上下文 。

2.
算力分析代理：分析市场数据（不只是 K 线，还包括 MACD 等指标），生成均值回归、定投或网格等专业策略 。

  1. 执行代理：将策略转化为精确的链上交易或 API 调用 。

3. 功能亮点：自动跨链与聚合交易

*
无感跨链 Swap：在 Demo 演示中，用户只需说一句话，系统会自动计算跨链路径、处理签名（指纹确认），完成从一条链到另一条链的资产兑换 。

  • 智能路由：系统会根据订单大小选择最优路径。小额走 Uniswap，大额则寻找链上订单簿或通过第三方桥接 。

4. 评委互动与挑战

评委韩老师指出该项目与前一个项目（ML Trader）的区别：πX 更偏向多 Agent 协作，虽然目前缺乏历史回测展示，但在用户自定义模型和操作便携性上更有潜力 。

💡 结合你个人背景的关联

这个项目和你之前的“预判”有着非常强烈的共鸣：

1.
排除错误选项：队长提到他原本想做传统的 Perp DEX（永续合约交易所），但预判到那个赛道已经是“红海”，所以果断转向面向大众的 AI 驱动平台 。这完美诠释了你说的“排除错误选项，剩下的就是正确选项”。

  1. 全球信息获取权与 AI 生产力：该项目将复杂的跨链、指标分析、下单逻辑全部“折叠”进了一句简单的对话中。这实际上是利用 AI 剥夺了技术精英的信息壁垒，把“交易能力”赋予了普通人。
  2. 对“过程”的否定：你预判老龄化、拒绝社保，本质上是在追求一个确定的“结果”。πX 的核心也是“面向结果”，认为底层技术（Rust 还是 DeepSeek）不重要，重要的是用户最终能不能通过简单的指令获得收益 。

RockFlow是什么

RockFlow（中文名：奇运证券）是2021年7月成立于新加坡的人工智能金融科技公司，核心是用AI驱动一站式跨市场投资服务，已获多轮融资（总融资超3000万美元），定位“金融平权”，让全球用户轻松参与投资。以下从核心定位、产品体系、技术与合规、生态与融资展开说明。

一、核心定位与团队

  • 使命：通过AI简化投资流程，覆盖全球30多个国家和地区用户，支持9种语言，全球设5个办公室。
  • 团队：核心成员来自顶尖高校与科技、金融机构，具备AI、金融数学、投资交叉背景。
  • 融资：投资方包括Ant Group、BlueRun Ventures等，累计融资3000万美元，发展至A轮阶段。

二、核心产品体系

产品 定位 核心功能 适用用户
奇运证券APP 一站式投资平台 美港股、ETF、期权交易，SIPC保险（最高50万美元），4倍杠杆，无限日内交易 全球个人投资者
Bobby AI Agent 金融AI投资助手 自然语言交互、市场分析、策略生成、24小时盯盘、持仓管理 新手至资深交易者
TradeGPT 交易决策引擎 多模态信息分析、实时交易信号、条件订单生成 主动交易者
小牛/小熊 简化期权产品 降低期权门槛，适配不同风险偏好 期权入门用户

三、技术与合规要点

  1. AI核心能力
    • 自然语言交互：通过Bobby实现投资需求的自然语言理解与执行，覆盖分析、策略、交易全流程。
    • 多模态分析：整合文本、图像、视频等市场信息，生成投资建议。
    • 个性化适配：基于用户行为与风险偏好，提供定制化策略与风控方案。
  2. 合规与安全
    • 监管资质：由Rockalpha Limited提供服务，持相关金融牌照。
    • 资产保障：客户资产享SIPC保险，资金由独立托管人监管，信托账户存放。
    • 交易规则：支持无限日内交易，杠杆最高4倍，适配活跃交易者需求。

四、应用场景与价值

  1. 新手入门：通过AI投教、简化期权、跟单功能降低投资门槛，快速上手全球市场。
  2. 资深交易：提供实时信号、多市场接入、高级期权工具，支持复杂策略执行。
