【GWDC会议笔记】HashKey Group

🏦 HashKey Group 是什么？

HashKey Group 是一家总部位于香港的数字资产金融集团，成立于 2018 年，专注于合规的 Web3 与区块链基础设施建设。

它是亚洲最早一批走“持牌合规路线”的加密金融机构之一。

🧱 它具体做什么？

HashKey 不是单一交易所，而是一个“集团矩阵”，核心业务包括：

1️⃣ 交易所
  • HashKey Exchange
  • 香港证监会（SFC）发牌的持牌虚拟资产交易平台
  • 面向专业投资者 + 现已向零售开放
  • 主打“合规、安全”

👉 类似香港版的 Coinbase，但更强调亚洲市场。

2️⃣ 投资机构
  • HashKey Capital
  • 专门做区块链 VC 投资
  • 投过很多基础设施项目、公链、DeFi 项目

👉 类似 a16z crypto 的亚洲版本。

3️⃣ 基础设施 & 技术服务
  • HashKey Cloud
  • 提供节点服务、Staking、技术解决方案
  • 面向机构客户

🏛 它的定位是什么？

一句话总结：

HashKey 是“香港最合规的加密金融集团之一”。

它代表的是：

  • 🇭🇰 香港政府支持的 Web3 金融路线
  • 🏦 传统金融 + 加密结合
  • 📜 强监管、持牌经营
  • 💰 面向机构资金

🌍 它是美系？中资？港资？

从地缘政治角度看：

  • 总部在香港
  • 强烈拥抱香港监管体系
  • 股东背景较多元（含传统金融资本）

它属于：

香港金融体系内的 Web3 旗舰玩家

不是草根 crypto 项目，而是金融资本推动型。

🧠 对你来说意味着什么？

Yuki，你现在在考虑：

  • Avalanche
  • Conflux
  • 长期协议层
  • 基础设施 vs SaaS

那 HashKey 属于哪一类生态？

👉 它是“金融合规生态”的核心节点。
👉 适合 RWA、合规资产发行、机构级基础设施。
👉 不太适合纯社区草根创新型玩法。

如果你的「金库协议」未来走 RWA + 合规路线，
HashKey 这种集团是潜在战略通道。

我们拆开讲清楚。

一、RWA 到底是什么？

RWA = Real World Assets（现实世界资产）

本质定义：

把链下的“可主张权益资产”映射到链上，让链上世界可以识别、交易、管理它。

关键词有三个：

  1. ✅ 现实世界存在
  2. ✅ 有法律权属
  3. ✅ 链上可验证或可流转

举例

典型 RWA
  • 国债
  • 房产
  • 供应链应收账款
  • 私募股权
  • 黄金
  • 碳排放额度

不是 RWA 的

  • 游戏道具
  • NFT 头像
  • 纯链上 Token
  • Meme 币

那是“原生链上资产”，不是 RWA。

二、RWA 是否一定要和 DeFi 绑定？

答案是：

❌ 不一定。

但现在大部分项目“选择”绑定 DeFi，是因为好讲故事。

我们分三种模型：

① RWA + DeFi（当前主流叙事）

例子：

  • 把美国国债 token 化
  • 放进链上借贷协议
  • 做收益抵押
  • 做稳定币底层资产

典型玩家：

  • MakerDAO
  • Centrifuge
  • Ondo Finance

这类项目的核心目标：

把现实资产变成 DeFi 的“收益来源”

因为 DeFi 需要“真实收益”。

② RWA + 合规金融（不碰 DeFi）

例如：

  • 香港持牌机构发行 token 化基金
  • 证券型代币（STO）
  • 合规私募基金上链

比如：

  • HashKey Group
  • HSBC 在香港做 token 化债券

这种模式：

本质是金融基础设施升级
不需要 DeFi
只是把登记系统从数据库换成链

③ RWA + 供应链 / 实物溯源（你更接近这个）

例如：

  • 商品一物一 NFT
  • 链上记录 ownership
  • 退货权利、保修权利
  • 二手流转分润

这类 RWA：

不为了金融杠杆
是为了“事实层可信”

这其实是：

资产状态上链，而不是资产融资上链

三、RWA 的核心本质

RWA 不等于 DeFi
RWA 不等于发币
RWA 不等于融资

RWA 本质是：

把“可主张权利”锚定到链上

关键问题只有一个：

链上记录，是否能改变现实世界的博弈？

如果能，就是 RWA。

四、你现在的“金库协议”算不算 RWA？

我们冷静判断。

你的模型是：

  • 每个商品 1 NFT
  • 首次扫码 = 权利激活
  • 售后权利上链
  • 二手流转可追溯

这是不是 RWA？

如果你只是记录数据 → ❌ 不是

如果 NFT 真正成为：

  • 售后凭证
  • 退货权利凭证
  • 分润权利凭证

那它就是：

权利型 RWA

而且是“微型 RWA”

不是金融大资产
而是“消费级权利资产”

五、市场误区

现在市场有个误区：

“只有能抵押借钱的才算 RWA”

这是 DeFi 视角。

但从更高层看：

所有链上与现实法律关系绑定的资产
都是 RWA

六、一个更本质的问题

你真正要问的是：

“我是不是应该把自己定义为 RWA 项目？”

战略上分三种说法：

  1. 对 Avalanche 说：我是 RWA
  2. 对供应链说：我是溯源系统
  3. 对金融机构说：我是合规资产底层

技术一样
叙事不同。

Web3 联盟数据库是什么

在 Web3 语境下，所谓的“联盟数据库”通常指的是：

联盟链（Consortium Blockchain）架构下的共享数据系统

它不是完全开放的公链数据库，而是：

多个机构共同维护，但只有被授权成员可以写入和验证数据的链上数据库。

一句话定义

Web3 联盟数据库 =
多方共治 + 有准入门槛 + 链上存证 + 权限控制

和公链的区别

类型 谁能写数据 谁能读数据 代表
公链 任何人 任何人 Ethereum
联盟链 被授权成员 通常可读或部分公开 Hyperledger
私链 单一机构 单一机构 企业内部链

Web3 联盟数据库的核心特征

① 多机构共识

不是一个公司控制
而是多个节点共同签名、记账。

例如：

  • 银行联盟
  • 供应链品牌联盟
  • 出版社版权联盟
② 数据不可篡改

虽然不是完全公开，
但写入后依然：

  • 有区块结构
  • 有哈希
  • 可溯源
③ 权限控制

不像公链人人能写。

通常采用：

  • 白名单节点
  • 身份认证
  • RBAC 权限系统

常见实现框架

  • Hyperledger Fabric
  • R3
  • Quorum

技术上它到底是什么？

本质上它是：

常用共识：

  • PBFT
  • Raft
  • Istanbul BFT

比公链快很多，
但牺牲了一部分去中心化。

为什么企业喜欢联盟数据库？

因为：

需求 公链 联盟数据库
数据可信 ✔ ✔
交易速度 慢 快
数据隐私 差 可控
法律合规 难 容易

举个你能理解的例子

如果：

  • 10 个奢侈品牌
  • 5 个跨境平台
  • 3 个鉴定机构

共同维护一个“真品数据库”，

但普通人不能随便写数据，

这就是一个典型的 Web3 联盟数据库。

它和你现在做的有什么关系？

如果你的“金库协议”：

  • 只允许品牌 mint NFT
  • 用户只能读取
  • 商家节点参与记账

那么它在结构上：

更接近公链 + 应用层权限控制
而不是严格意义的联盟链

但如果未来你：

  • 限制节点参与
  • 多品牌共治
  • 不完全开放

那就会演化成：

联盟式链上数据库

核心本质

Web3 联盟数据库不是一个新数据库类型，
而是：

一种“多机构共同治理的数据架构模型”。

PHAROS是什么

🚀 Pharos Network 是什么？

Pharos 是一个新兴的区块链基础设施项目，定位为高性能、模块化的公链网络，主要面向 Web3 应用开发者。它通常出现在黑客松、开发者生态活动中（比如你之前提到的 APAC 黑客松）。

🧱 它本质是什么？

可以理解为：

一个 Layer 1 公链项目，强调性能与可扩展性，吸引开发者生态。

它的目标通常包括：

  • ⚡ 高 TPS（高吞吐）
  • 🧩 模块化架构
  • 🔗 兼容 EVM（多数新链都会做）
  • 🧠 面向 DeFi / RWA / 基础设施类项目

🧭 它和 Avalanche、Conflux 是什么关系？

你最近一直在研究 Avalanche、Conflux 生态战略对吧？

可以这样理解：

公链 生态定位 特点
Avalanche 成熟高性能 L1 子网 + 强资本支持
Conflux 中国背景公链 关系密集型生态
Pharos 新兴链 早期生态，需要项目

Pharos 属于那种：

🌱 生态早期阶段，急需有战略意义的项目进入。

这类链往往：

  • 资金扶持力度大
  • 愿意给早期项目资源曝光
  • 但长期确定性比 Avalanche 低

🧠 从你“金库协议”的角度看

你现在的项目是：

协议层基础设施 + 实物资产追踪 + NFT + 退货倒计时 + 双码设计

你要问的其实是：

❓ Pharos 值不值得长期绑定？

这要看它：

  • 是否支持 EVM（你 Solidity 能否直接部署）
  • 出块速度 / 费用情况
  • 是否有 RWA 或溯源类战略方向
  • 是否给长期扶持（不是黑客松拿钱就走）

🎯 一句话总结

Pharos 是一个 正在建立开发者生态的 Layer 1 区块链项目，
类似于 Avalanche 早期阶段。

如果你：

  • 想拿早期生态红利 → 可以参与
  • 想构建长期协议层 → 要评估其生态持续性

一、Layer 1 的技术门槛到底高不高？

结论：发链门槛低，真正做出生态门槛极高

今天做 L1 的路径大概有三种：

  2. 改个共识参数

  3. 发个 Token

  4. 做个官网 + 黑客松

——你就可以“拥有一条公链”。

所以：

🚪 发链门槛低
🧠 真正做出生态门槛极高

二、Layer 1 死了多少？

非常多。

下面给你一个真实历史感。

🔥 2017–2018 ICO 时代

当年号称“以太坊杀手”的：

  • EOS Network
  • NEO
  • Qtum
  • Stratis

现在基本都已经边缘化。

🔥 2020–2022 公链爆发期

  • Terra（直接崩盘）
  • Harmony（被黑客掏空）
  • Near Protocol（生态收缩）
  • Fantom（叙事消退）

能真正站稳的，其实很少。

三、为什么 L1 很容易死？

你在黑客松里已经感受到一个现象：

项目为了奖金来
拿完钱就走
生态没有真实用户

L1 想活下来，必须有：

  1. 💰 资金持续性
  2. 👥 真正留在链上的开发者
  3. 📈 有真实用户的应用
  4. 🧲 形成网络效应

只要其中一个断裂，链就慢慢变成“空壳链”。

四、真正活下来的 L1 有哪些？

全球范围真正稳固的极少数：

  • Bitcoin
  • Ethereum
  • Solana
  • Avalanche

大概就这个级别。

其他都在挣扎。

五、所以你现在的直觉是对的

你开始发现：

“Layer 1 是不是已经供给严重过剩？”

答案是：是的。

现在不是缺链。

现在缺的是：

  • 真正的协议层创新
  • 真实资产落地
  • 真正解决商业问题的系统

而不是再发一条链。

六、真正难的不是 Layer 1

真正难的是：

🧠 构建“不可替代的协议层”

你现在做的“金库协议”，
如果它是：

  • 跨链可移植
  • 不依赖单一 L1
  • 解决真实退货/溯源/确权问题

那它的价值远高于单条链。

  1. 它是否有独特共识创新？
  2. 它是否有真实用户？
  3. 它是否有长期资本？

如果三个都没有：

那它只是“黑客松链”。

豆包的回答

Layer 1 区块链项目的门槛并不低，且死亡率极高。你在黑客松看到的“快速出链”，大多是低代码/分叉/模块化的快速原型，与真正能长期存活的独立公链完全是两回事。

一、Layer 1 门槛：看起来低，实则极高

1. 表面“低门槛”（黑客松常见）

  • 快速分叉/模块化：用 Cosmos SDK、Substrate、EVM 兼容框架（如 BSC、Polygon 前身），几周就能出一条链。
  • 共识/虚拟机复用：直接用 PoS、DPoS、EVM 等成熟组件，不用从零写共识与 VM。
  • 黑客松模板：很多赛事提供一键发链工具，侧重 Demo 与叙事，不考验长期运维。

2. 真实“高门槛”（存活必备）

  • 技术深度：从零开发需精通 P2P、共识、VM、密码学、网络安全，团队至少 3–10 人资深工程师，周期 6 个月以上。
  • 安全与运维：持续抗 51% 攻击、漏洞修复、节点升级、灾备，成本极高。
  • 生态冷启动：吸引验证者/矿工、开发者、用户、资金，形成正循环；多数链死在“有链无生态”。
  • 代币经济与价值捕获：设计可持续的通胀/通缩、质押、手续费分配；否则代币归零、团队跑路。
  • 合规与监管：全球监管趋严，合规成本与风险巨大。

二、Layer 1 死亡率：绝大多数已死或僵尸化

  • ICO 时代（2017–2018）：数百条 L1 上线，90%+ 已死亡/边缘化。
  • 2021–2022 牛市：Solana、Avalanche、Terra 等爆发后，大量新 L1 跟风，70%+ 已死或 TVL 趋近于零。
  • 现状：仅 BTC、ETH、Solana、BNB Chain、Avalanche、Cosmos 等少数头部存活；排名 20 以外的 L1 几乎无真实价值。

三、典型“已死/僵尸”Layer 1 项目（按死亡原因）

1. 算法稳定币崩盘（最惨烈）

  • Terra (LUNA/UST)：2022 年 5 月 UST 脱钩，LUNA 从 80+ 美元归零，链虽在但生态彻底死亡。
  • Iron Finance (IRON/TITAN)：2021 年算法稳定币崩盘，链与代币归零。

2. 技术/安全事故致死

  • Saga：2026 年 1 月跨链漏洞导致 700 万美元损失，团队停止运营。
  • EOS：虽未归零，但社区分裂、开发停滞、TVL 极低，沦为僵尸链。
  • Tezos：早期明星，技术迭代慢、生态冷清，长期边缘化。
  • Algorand：技术优秀但生态冷启动失败，用户与开发者流失严重。

3. 生态枯竭/无价值捕获

  • NEO：“中国以太坊”，生态萎缩、无杀手级应用，沦为历史。
  • Waves：早期公链，DeFi/NFT 均无起色，TVL 几乎为零。
  • Zilliqa：主打分片，性能与生态均被超越，长期低迷。
  • Harmony：跨链桥被盗后一蹶不振，生态崩溃。

4. 团队跑路/项目放弃

  • Aeternity：团队解散、代码停更，链已无人维护。
  • Lisk：侧链概念先驱，开发停滞、社区消亡。

5. Cosmos 生态内死亡案例

  • Pryzm：完全关闭。
  • Comdex、Kujira：相继倒下，带走子项目，DeFi 链条断裂。
  • Mars Protocol：2026 年漏洞后宣布 3 月前停止运营。

四、死亡核心原因（通用规律）

  1. 无差异化：技术、性能、生态无护城河，被头部碾压。
  2. 生态空心化：有链无 DApp、无用户、无 TVL，沦为“空气链”。
  3. 代币经济崩溃：通胀失控、无价值捕获、团队套现、流动性枯竭。
  4. 安全事故：漏洞、51% 攻击、跨链被盗，一次性致命。
  5. 监管与合规：被监管打击、交易所下架、资金出逃。
  6. 团队与资金：熊市资金断裂、团队流失、放弃维护。

五、结论与黑客松视角

  • 黑客松≠真实世界：你看到的快速 L1 是 Demo，不是可存活的公链。
  • L1 是“基建级战争”：只有解决真实需求、建立强生态、具备安全与经济可持续性的链才能活。
  • 未来趋势：模块化 L1、AppChain、L2 成为主流，独立通用 L1 的机会窗口已关闭；新 L1 必须垂直化（如 DeFi、游戏、AI）才有生存空间。
