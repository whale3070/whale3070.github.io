核心函数

我来帮你逐个讲解你这个 ERC20 合约里的核心函数，以及如何在 Remix 的 Deployed Contracts 页面里测试它们。我们按功能来拆解，尽量结合操作示例，直观易懂。

1️⃣ balanceOf

作用：

  • 查询某个地址的 代币余额。
  • 不修改区块链状态，只是读取。

使用方法：

  1. 部署合约后，在 Deployed Contracts 找到 balanceOf 函数。
  2. 输入一个地址（比如部署者地址）。
  3. 点击 call → 返回该地址的代币数量。

注意：

  • 如果还没有 mint 过代币，余额是 0。

2️⃣ transfer

作用：

  • 将自己账户里的代币转给别人。
  • 调用者是 msg.sender（也就是发送者），只会扣自己的余额。

使用方法：

  1. 在 Deployed Contracts 找到 transfer。

  2. 输入：

    • to → 对方地址
    • amount → 代币数量

  3. 点击 transact（这里会修改链上状态）。

  4. 使用 balanceOf 可以验证转账是否成功。

注意：

  • 余额不足会 revert → 报 "balance too low"。
  • 不能给 0 地址转账 → 报 "to zero"。

3️⃣ approve

作用：

  • 授权别人（spender）可以 从你的账户花费一定数量的代币。
  • 并不会真正转币，只是设置 allowance。

使用方法：

  1. 找到 approve 函数。

  2. 输入：

    • spender → 被授权者地址
    • amount → 允许花费的代币数量

  3. 点击 transact → 会触发 Approval 事件。

  4. 可以通过 allowance 查看授权额度。

注意：

  • 没有余额也可以 approve，只是别人 transferFrom 时会失败。

4️⃣ allowance

作用：

  • 查询 owner 给 spender 授权了多少代币。

使用方法：

  1. 找到 allowance。

  2. 输入：

    • owner → 授权人地址
    • spender → 被授权人地址

  3. 点击 call → 返回剩余可用额度。

5️⃣ transferFrom

作用：

  • 被授权者（msg.sender）从别人账户 from 转代币到 to。

  • 需要：

    • from 已经 approve msg.sender 足够额度
    • from 有足够余额

使用方法：

  1. Alice approve Bob 50 个 token
  2. Bob 调用 transferFrom(Alice, Charlie, 20)
  3. _allowances[Alice][Bob] 会减少 20
  4. _balances[Alice] 扣 20，_balances[Charlie] 加 20

注意：

  • 余额或授权不足会 revert
  • msg.sender = 调用这个函数的人

6️⃣ _mint

作用：

  • 铸造新的代币给某个地址
  • 增加 totalSupply
  • 增加 to 的余额

使用方法（Deployed Contracts 不能直接调）：

  • _mint 是 internal，只能在合约内部调用（比如部署时或写一个 public mint 函数）。
  • 如果想测试，需要在合约里加一个 public 包装函数：
  • 然后在 Deployed Contracts 调用 mint，自己给自己增加代币。

7️⃣ _burn

作用：

  • 销毁代币
  • 扣除 from 的余额
  • 减少 totalSupply

使用方法（Deployed Contracts 不能直接调）：

  • 同 _mint，需要一个 public 包装函数：
  • 然后调用 burn，会减少你账户的代币和总量。

✅ ② transferFrom 为什么需要 allowance，而 transfer 不需要？

🔥 ERC20 标准设计 transferFrom 的真正原因：授权第三方代你花钱
